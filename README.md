# 🔬 Evaluation Drift Monitor (EDM)

> **"AI systems trust stale evaluation datasets that no longer reflect real-world usage."**

EDM is a professional-grade observability tool designed to track the divergence between your static benchmarks and evolving production usage patterns. It ensures that your LLM evaluations remain relevant and reliable by identifying **Distribution Drift** and **Semantic Shift**.

---

## 🧠 Researcher's Mindset (PhD Perspective)

### 1. Correctness & Methodology
EDM uses **Embedding Centroid Drift (ECD)**. We map prompts into a high-dimensional semantic space $\mathbb{R}^d$. 
- **Baseline $(\mathcal{D}_{base})$:** The centroid $\mu_{base} = \frac{1}{|\mathcal{D}_{base}|} \sum \vec{v}_i$.
- **New Run $(\mathcal{D}_{new})$:** The centroid $\mu_{new} = \frac{1}{|\mathcal{D}_{new}|} \sum \vec{u}_j$.
- **Drift Metric:** Cosine Distance $1 - \frac{\mu_{base} \cdot \mu_{new}}{\|\mu_{base}\| \|\mu_{new}\|}$.

### 2. Limitations
- **Latency-Accuracy Tradeoff:** Real-time drift detection requires high-throughput embedding generation which may introduce cost/latency overhead.
- **Subjectivity:** Drift doesn't always imply degradation (e.g., users asking more advanced questions is "Good Drift").

### 3. Assumptions
- **Semantic Mapping:** We assume that local density shifts in the prompt embedding space accurately reflect changes in user intent.
- **Static Quality:** We assume the baseline benchmark was "correct" at the time of creation.

## 🤖 AI Logic: Why and How?

### Which AI?
We utilize **Semantic Embedding Models** (e.g., `all-MiniLM-L6-v2`) and **LLM-as-a-judge** (GPT-4o-mini).

### Why not other AI?
- **Keyword Matching (Lexical):** Fails to capture intent. "How to pay" vs "Payment methods" are lexically different but semantically identical.
- **PCA/t-SNE:** Good for visualization, but poor for quantitative alerting without high-dimensional preservation.
- **Standard Statistical Tests (K-S Test):** Only work on 1D numerical data. Prompt drift is multi-dimensional and semantic.

### Simulation Logic
EDM simulates a production environment by comparing:
1.  **Golden Dataset:** Your fixed evaluation set.
2.  **Shadow Run:** Recent session logs or a secondary evaluation run.
3.  **Cross-Validation:** Calculating the JS-Divergence across categorical labels generated by a classifier LLM.
